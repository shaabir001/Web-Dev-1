By completing this exercise, you will:

  • Become acquainted with the problem of source control.
  • Understand the solution offered by GitHub for collaborative development.
  • Consider the social impacts of using collaborative tools like GitHub.

Introduction

Version control is a powerful tool for developing any project or product. If you are collaborating with one or more other people, it becomes even more important. Have you ever found it difficult to keep papers well organized? Imagine 35 people working on the same document at the same time. GitHub provides a solution to these challenges through effective version control and collaboration features like pull requests.

Materials

  • Computer with Internet browser and GitHub client software.
  • GitHub individual account and membership in an organizational account's team.

Procedure

Part III: Share Your Changes

  1. After committing all your changes, you can now Push to origin. This will make your changes available on GitHub.com.
  2. It's now time to make your changes known to the top master. Create a Pull Request so your changes can be merged into the original code.
  3. Click on the branch tab and select Create pull request or use the shortcut cmd+R.
  4. This will take you to GitHub.com where you will perform the following steps to pull your changes into the original code.

    5. Important: Make sure that the correct forks are selected. You want your master head fork on the right and the WebDev-ShakoHS/WebDev repository on the left. If you see any errors, fix them before submitting your changes. Once everything looks good, click the green Create pull request button.

  5. Give your pull request some descriptive information, and then you can finally create the pull request!

    6. Your teacher will now receive a notification that someone has code they would like to merge into the original repository. If everything was done correctly, your class's code will be easily merged into the class repo's current version (the head commit) on the master branch.
