What are CSS Animations?

CSS animations allow you to animate transitions between one CSS style to another. This can bring your web page to life with smooth effects and transitions. Animations can be triggered by user interactions or can run automatically.

What are CSS Card Animations?

CSS card animations can be used to animate images or boxes on a website. CSS card animations methods are often used to create interactive experiences when users click on items. With CSS card animations, you can add visual effects and transitions like flipping or changing the rotation of the card!

1. Creating a gallery.css file

We will start from the top and work our way down. You can start by creating a new .css file and name it gallery.css The file is already linked in head of your .html file, so make sure you name it with correct spelling!

2. Creating a ID!

Copy and paste the code shown below into your gallery.css file

#gallery { font-family: "Orbitron", sans-serif; line-height: 1.5em; text-align: center; }

The most outer "div"(line #1 from above) has the id "gallery" so we will start by giving that "div" 3 basic styles; a font, a specific line height, and alignment of the text to the center.

The added code should look like this:

3. Creating a container!

Copy and paste the code from below into your gallery.css file

.container { display: inline-block; margin: 3%; perspective: 1000px; }

This is the "div" that has the class "container". We want the items to have a set width and height and be able to give them a margin and padding so we change the display to inline-block. When choosing a margin we can choose from a large variety of units, for this example we will use percentage. Not much will really change except the margin size so an example won't be shown here.

4. Creating a flipper!

Copy and paste the code from below into your gallery.css file

.flipper { position: relative; transition: 0.6s; transform-style: preserve-3d; }

Moving to the "div" that has the class "flipper". Since the "flipper" is a child of the container we will use the relative position. Using the relative position will mean we can position the "flipper" div relative inside of the "container" div. The relative position is very straightforward, it just means that the element is positioned relative to its normal position. Transition will determine how long our animation will take, in this case 6 seconds. This styling will not change anything at all for the time being so an example won't be provided.

5. Adding front and back cards!

Copy and paste the code from below into your gallery.css file.

.front, .back { position: absolute; top: 0; left: 0; width: 250px; height: 320px; box-shadow: 0 4px 8px 0 rgba(0, 0, 0, 0.2), 0 6px 20px 0 rgba(0, 0, 0, 0.19); backface-visibility: hidden; } .front{ z-index:2; } .back{ transform:rotateY(180deg); }

The last 2 div's we will style are the "front" and "back". We will use 3 different selectors 1 we will combine styling for the "front" and "back" and 1 each for seperate "front" and "back" styling. Backface-visibility is yet another obscure property. When we turn an object around or transform it we can make sure we don't see the "backside" of that object. In this case think of a playing card, when you flip it over you reveal the number/color/suit, when we hide the visibility we see nothing. 7. Seperating and adding animation to the cards!

Copy and paste the code from below into your gallery.css file.

.container, .front, .back { width: 250px; height: 320px; } .container:hover .flipper { transform: rotateY(180deg); }

To make the cards separated and not on top of each other let's give them a specific size. If you want to change the size here is where you can do that. Using the hover pseudo class here will make just a space between .container:hover and .flipper. This means that .flipper needs to be child of .container in order to work.
